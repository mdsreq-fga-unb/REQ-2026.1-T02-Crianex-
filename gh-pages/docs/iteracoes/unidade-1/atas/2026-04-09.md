## Discussões e Decisões

### 1. Contextualização do Projeto

O projeto **Crianex** foi oficialmente aprovado pelo professor orientador. Trata-se do desenvolvimento de uma plataforma para a Crianex, uma _Software House_ B2B, com foco em resolver dois problemas centrais:

- **Problema:** Gestão descentralizada de produtos e baixa visibilidade do portfólio no mercado.
- **Solução:** Uma plataforma unificada composta por duas frentes:
  - **Área Administrativa:** Gestão de logs, status de projetos e alocação de pessoas em tempo real (acesso via login).
  - **Vitrine Digital:** Exposição do portfólio para aumento de engajamento (acesso público).

### 2. Definições de Engenharia de Software

Ficou acordado que o projeto seguirá a metodologia **FDD (Feature Driven Development)** com gestão de fluxo **Scrumban Enxuto**, dentro de um ciclo de vida **Iterativo e Incremental**.

#### 2.1 Metodologias

| Dimensão          | Metodologia                          | Detalhes                                                                                                                                                                                                            |
| ----------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Gestão de Produto | **FDD** (Feature Driven Development) | Orientação a funcionalidades. As 5 etapas serão seguidas: Desenvolver modelo geral → Construir lista de funcionalidades → Planejar por funcionalidade → Projetar por funcionalidade → Construir por funcionalidade. |
| Gestão de Fluxo   | **Scrumban Enxuto**                  | Iterações com duração variável. Quadro visual (Kanban Board) gerenciado no GitHub Projects. Sync diário assíncrono via Discord.                                                                                     |
