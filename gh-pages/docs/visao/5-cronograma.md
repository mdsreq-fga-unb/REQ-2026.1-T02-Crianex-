## 5.1 Visão Geral do Cronograma

O projeto é organizado em **sprints de 2 semanas**, alinhadas ao calendário acadêmico da disciplina de Requisitos de Software (REQ-2026.1). O projeto possui 3 unidades de entrega, cada uma correspondendo a um conjunto de sprints com objetivos incrementais.

### Marcos Principais

| Kickoff do Projeto | 09/04/2026 | Reunião de alinhamento e definição de escopo |

| Início Sprint 1 | 14/04/2026 | Início do desenvolvimento do MVP |
| Entrega — Unidade 2 | A definir | Incremento do produto com funcionalidades MVP |
| Entrega — Unidade 3 | A definir | Produto completo com vitrine digital e área administrativa |
