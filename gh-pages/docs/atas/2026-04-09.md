## Discussões e Decisões

### 1. Contextualização do Projeto

O projeto **Crianex** foi oficialmente aprovado pelo professor orientador. Trata-se do desenvolvimento de uma plataforma para a Crianex, uma *Software House* B2B, com foco em resolver dois problemas centrais:

- **Problema:** Gestão descentralizada de produtos e baixa visibilidade do portfólio no mercado.
- **Solução:** Uma plataforma unificada composta por duas frentes:
    - **Área Administrativa:** Gestão de logs, status de projetos e alocação de pessoas em tempo real (acesso via login).
    - **Vitrine Digital:** Exposição do portfólio para aumento de engajamento (acesso público).

### 2. Definições de Engenharia de Software

Ficou acordado que o projeto seguirá uma abordagem **Ágil** com ciclo de vida **Iterativo e Incremental**.

#### 2.1 Metodologias

| Dimensão | Metodologia | Detalhes |
|----------|-------------|----------|
| Gestão de Produto | **FDD** (Feature Driven Development) | Orientação a objetos e foco em funcionalidades. As 5 etapas serão seguidas: Desenvolver modelo geral → Construir lista de funcionalidades → Planejar por funcionalidade → Projetar por funcionalidade → Construir por funcionalidade. |
| Gestão de Equipe | **Scrumban** (Scrum + Kanban) | Sprints de duas semanas. Quadro visual (Board) gerenciado no GitHub Projects. |

#### 2.2 Papéis da Equipe

| Papel | Responsabilidade |
|-------|-----------------|
| Product Owner (PO) | Interface com o cliente (Otávio / Vitor) |
| Tech Lead | Coordenação técnica e cronograma |
| QA (Quality Assurance) | Revisão de Pull Requests (PRs) |
| Dev Team | Desenvolvimento das funcionalidades |

#### 2.3 Stack Tecnológica *(a confirmar com o cliente)*

| Camada | Tecnologia |
|--------|-----------|
| Frontend | Next.js, Svelte ou Vue (a definir) |
| Backend | NestJS (Node.js) |
| Banco de Dados | Supabase / PostgreSQL |
| Infraestrutura | Kubernetes (em avaliação) |

### 3. Riscos e Comunicação

- **Risco identificado:** Sobrecarga de membros que cursam a disciplina de IHC (Interação Humano-Computador), podendo impactar os prazos.
- **Canais de comunicação:** Discord organizado por categorias — Backlogs, Knowledge, Gravações, Arquitetura.
- **Monitoria:** O TechLead entrará em contato com a monitora para validar a questão dos commits individuais no GitHub Pages para fins de avaliação.
