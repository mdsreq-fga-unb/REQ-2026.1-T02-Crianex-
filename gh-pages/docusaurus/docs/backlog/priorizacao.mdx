## Matriz Valor × Esforço

Cada ponto é uma feature, posicionada pelos valores reais de **VB** (eixo Y) e **ES** (eixo X). A região verde (alto valor, baixo esforço) delimita o **MVP**.

<ValueEffortMatrix />

## Método de Cálculo

```
IP = VB / ES   (aplicado diretamente à Feature)

IP ≥ 1,50      → Alta prioridade   (Q1)
IP 1,00–1,49   → Média prioridade  (Q2)
IP < 1,00      → Baixa prioridade  (Q3/Q4)
```

| Sigla | Descrição |
| ----- | --------- |
| **VB** | Valor de Negócio — impacto direto nos OEs |
| **ES** | Esforço de implementação da Feature completa |
| **IP** | Índice de Prioridade = VB / ES |

### Por que top-down?

A abordagem anterior calculava o IP de cada RF e **agregava** para estimar a feature (bottom-up) — a média dos RFs nivela diferenças internas e não representa o valor real entregue pela feature como unidade. No **top-down**, priorizamos as **Features** diretamente (unidade de entrega e de decisão); o cálculo de RFs serve apenas para ordenar a implementação _dentro_ de uma feature já priorizada.

## Critérios

<Tabs>
<TabItem value="valor" label="Critérios de VALOR (VB)">

`VB = (Impacto de Negócio × 3) + (Frequência de Uso × 3) + (Valor ao Usuário × 2) + (Impacto Estratégico × 2)`

| Critério | Peso | Descrição |
| -------- | :--: | --------- |
| **Impacto de Negócio** | 3 | Contribuição direta aos objetivos estratégicos/operacionais (produtividade, centralização, valor comercial) |
| **Frequência de Uso** | 3 | Com que frequência a feature será usada no dia a dia |
| **Valor percebido pelo Usuário** | 2 | Quanto melhora experiência, praticidade e eficiência percebida |
| **Impacto Estratégico** | 2 | Diferenciais estratégicos, crescimento futuro e posicionamento |

</TabItem>
<TabItem value="esforco" label="Critérios de ESFORÇO (ES)">

`ES = (Complexidade × 3) + (Dependências × 2) + (Risco Técnico × 2) + (Tempo × 1)`

> A complexidade considera **RFs e RNFs** — os RNFs impõem restrições de qualidade (segurança, desempenho, conformidade) que aumentam o esforço real.

| Critério | Peso | Descrição |
| -------- | :--: | --------- |
| **Complexidade técnica** | 3 | Dificuldade considerando lógica dos RFs, integrações, persistência e restrições dos RNFs |
| **Dependências** | 2 | Dependência de outros módulos/serviços para funcionar |
| **Risco técnico** | 2 | Probabilidade de falhas ou inconsistências no desenvolvimento |
| **Tempo de implementação** | 1 | Esforço temporal para desenvolver, testar e validar |

</TabItem>
</Tabs>
