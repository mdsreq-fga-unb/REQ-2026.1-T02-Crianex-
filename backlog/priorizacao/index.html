Priorização de Features — Crianex Hub

Priorização top-down: a unidade de decisão é a Feature, priorizada por IP = VB / ES. As features com melhor retorno relativo formam o MVP.

Matriz Valor × Esforço​

Cada ponto é uma feature, posicionada pelos valores reais de VB (eixo Y) e ES (eixo X). Duas linhas de corte dividem o plano em 4 quadrantes; o Quadrante I (alto valor + baixo esforço, canto superior-esquerdo) é o único pintado em verde e reúne exatamente as features do MVP.

Carregando matriz…
QuadrantePosiçãoSignificado
I — MVPAlto valor / Baixo esforçoMaior retorno relativo — prioridade máxima, compõe o MVP
IIAlto valor / Alto esforçoBoas candidatas, mas exigem mais esforço — reavaliar na próxima iteração
IIIBaixo valor / Baixo esforçoGanhos rápidos de baixo impacto — fazer se houver capacidade ociosa
IVBaixo valor / Alto esforçoPior retorno relativo — não priorizar
F08 é exceção, não erro de cálculo

F08 cai fora do Quadrante I (IP 1,31) e por isso não é classificada como MVP pela matriz. Ainda assim, entra na IT2 por dependência lógica com F07 (o histórico de notificações pressupõe templates existentes) — ver observação na tabela de features do MVP.

Método de Cálculo​

IP = VB / ES   (aplicado diretamente à Feature)

IP ≥ 1,50      → Alta prioridade   (Quadrante I — MVP)
IP 1,00–1,49   → Média prioridade  (Quadrante II)
IP < 1,00      → Baixa prioridade  (Quadrantes III/IV)
SiglaDescrição
VBValor de Negócio — impacto direto nos OEs
ESEsforço de implementação da Feature completa
IPÍndice de Prioridade = VB / ES

Por que top-down?​

A abordagem anterior calculava o IP de cada RF e agregava para estimar a feature (bottom-up) — a média dos RFs nivela diferenças internas e não representa o valor real entregue pela feature como unidade. No top-down, priorizamos as Features diretamente (unidade de entrega e de decisão); o cálculo de RFs serve apenas para ordenar a implementação dentro de uma feature já priorizada.

Critérios​

  • Critérios de VALOR (VB)
  • Critérios de ESFORÇO (ES)

VB = (Impacto de Negócio × 3) + (Frequência de Uso × 3) + (Valor ao Usuário × 2) + (Impacto Estratégico × 2)

CritérioPesoDescrição
Impacto de Negócio3Contribuição direta aos objetivos estratégicos/operacionais (produtividade, centralização, valor comercial)
Frequência de Uso3Com que frequência a feature será usada no dia a dia
Valor percebido pelo Usuário2Quanto melhora experiência, praticidade e eficiência percebida
Impacto Estratégico2Diferenciais estratégicos, crescimento futuro e posicionamento

ES = (Complexidade × 3) + (Dependências × 2) + (Risco Técnico × 2) + (Tempo × 1)

A complexidade considera RFs e RNFs — os RNFs impõem restrições de qualidade (segurança, desempenho, conformidade) que aumentam o esforço real.

CritérioPesoDescrição
Complexidade técnica3Dificuldade considerando lógica dos RFs, integrações, persistência e restrições dos RNFs
Dependências2Dependência de outros módulos/serviços para funcionar
Risco técnico2Probabilidade de falhas ou inconsistências no desenvolvimento
Tempo de implementação1Esforço temporal para desenvolver, testar e validar

Validação com o Cliente​

Os scores de VB foram derivados a partir dos registros de vídeos de reuniões e conversas com o cliente ao longo do projeto — o histórico de interações com Otávio fundamentou cada nota de 1 a 5 nos critérios de valor. Ao final, os resultados consolidados foram apresentados e validados diretamente com o cliente via WhatsApp.

O ES (Esforço) não passa por validação com o cliente: é estimado internamente pela equipe de desenvolvimento, que avalia complexidade técnica, dependências, risco e tempo.
